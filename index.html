Password Cracker

Who was it for?

This project was done for CS361 - Intro to Computer Security.

What was it?

It was a project done in Java to implement a password cracker when the password is encrypted with Jcrypt.

When was it?

In the summer of 2015.

Where was this?

This was done in Austin, TX at the University of Texas; more specifically at the GDC computer lab.

Why was it done?

It was a project to help us understand how someone would be able to write a simple program to crack passwords associated with accounts on a computer if they were using Jcrypt (a popular encryption). Given a dictionary of words and a log file of the encrypted passwords with the users, we use Jcrypt to help reverse engineer the encrypted passwords back to plaintext.

How was it done?

This project was done using Sublime Text as our preferred choice of text editor on a Linux machine (gnome). For testing, we just fed in different input files (encryptions and their plain text counter parts) and see how many passwords we crack successfully.
